📊 Tóm tắt

p-table emit onLazyLoad khi:

Trigger	Có gọi API
Render lần đầu	✅
Pagination	✅
Sort column	✅
Filter column	✅
Reset table	✅
Render lại table	✅



1️⃣ Khi table render lần đầu

Ngay khi component hiển thị lần đầu.

<p-table
  [value]="data"
  [lazy]="true"
  (onLazyLoad)="loadData($event)">
</p-table>

Flow:

Table render
   ↓
PrimeNG detect lazy mode
   ↓
emit onLazyLoad
   ↓
loadData(event)

Event trả về dạng:

{
  first: 0,
  rows: 10,
  sortField: null,
  sortOrder: null,
  filters: {}
}
2️⃣ Khi user đổi trang (pagination)

Khi dùng paginator:

[paginator]="true"

User bấm:

page 1 → page 2

p-table sẽ emit:

onLazyLoad

event ví dụ:

{
  first: 10,
  rows: 10
}
3️⃣ Khi sort column

Nếu column có:

<p-column field="name" [sortable]="true"></p-column>

User click header:

Name ↑
Name ↓

p-table sẽ emit onLazyLoad

event:

{
  sortField: "name",
  sortOrder: 1
}
4️⃣ Khi filter column

Nếu có filter:

<p-column field="name" filter></p-column>

User nhập filter → emit onLazyLoad.

event:

{
  filters: {
    name: { value: "abc", matchMode: "contains" }
  }
}
5️⃣ Khi reset table / change state

Một số action cũng trigger:

gọi table.reset()

thay đổi [rows]

thay đổi [first]

thay đổi [filters]

thay đổi [sortField]

6️⃣ Khi table bị destroy và render lại

Ví dụ:

<div *ngIf="showTable">
  <p-table ...>
</div>

Nếu:

showTable = false → true

table render lại → onLazyLoad.






Các trường hợp chắc chắn gây onLazyLoad
1️⃣ Render table lần đầu

Khi component hiển thị lần đầu.

Điều kiện:

[lazy]="true"

Flow:

p-table init
   ↓
PrimeNG detect lazy mode
   ↓
emit onLazyLoad

Event:

{
  first: 0,
  rows: rows
}
2️⃣ Đổi trang (pagination)

Do bạn bật:

[paginator]="paginator"

Khi user click:

page 1 → page 2

PrimeNG sẽ emit:

onLazyLoad({
  first: 10,
  rows: 10
})
3️⃣ Thay đổi số dòng mỗi trang

Do bạn có:

[rowsPerPageOptions]="rowsPerPageOptions"

User đổi:

10 rows → 20 rows

PrimeNG emit:

onLazyLoad

Event:

{
  first: 0,
  rows: 20
}
4️⃣ Sort column

Bạn có:

[sortMode]="'single'"
(onSort)="onSort($event)"

Nếu column có:

pSortableColumn

Khi user click header:

Name ↑
Name ↓

PrimeNG sẽ:

emit onSort
emit onLazyLoad

Event:

{
  sortField: "name",
  sortOrder: 1
}
5️⃣ Khi bạn thay đổi first từ code

Bạn bind:

[first]="first"

Nếu trong code:

this.first = 0;

table sẽ:

update state
→ emit onLazyLoad
6️⃣ Khi thay đổi rows

Bạn bind:

[rows]="rows"

Nếu:

this.rows = 20;

PrimeNG sẽ reload:

onLazyLoad
7️⃣ Khi reset table

Ví dụ:

this.table.reset();

PrimeNG sẽ:

first = 0
filters = null
sort = null
→ emit onLazyLoad
Những cái KHÔNG gây onLazyLoad

Các property này không trigger:

[value]
[(selection)]
(selectionChange)
dataKey

Ví dụ:

[(selection)]="selectedRows"

chỉ update selection, không reload data.

Tóm tắt toàn bộ
Trigger	Có gọi onLazyLoad
Table render lần đầu	✅
Pagination	✅
Rows per page change	✅
Sort column	✅
Change first	✅
Change rows	✅
Reset table	✅
Change selection	❌
Change value	❌
